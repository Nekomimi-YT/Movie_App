Movie Info App

Objective: To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Endpoint Information
Business Logic URL HTTP Method Request Body Data Format Response Body Data Format
Return all movies to the user /movies GET None A JSON object holding data about all the movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/:Title GET None A JSON object holding all the data about requested movie title
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/Genre/:genreName GET None A JSON object holding the data for the genre type (description) and more info about requested genre description
Return data about a director (bio, birth year, death year) by name /movies/Director/:directorName GET None Return a JSON object with data about a movie's director

Allow users to update their user info (username) /users PUT None A text message indicating that the userName has been updated
Allow users to add a movie to their list of favorite movies /users/:_id/:Title PUT None A text message indicating that the new favorite movie has been added.
Allow users to delete a movie from their list of favorite movies /users/:_id/:Title DELETE None A text message indicating that movie has been deleted.
Allow existing users to deregister by username /users/:Username DELETE None A text message showing that a user email has been removed.
